Synara assumes you live in it for hours a day, so most of the surface is tunable.

## Theme pack editor

Tune the UI colors directly in settings. Shared theme tokens keep the sidebar, composer, transcript, and controls in sync, and terminal fonts and colors follow the active theme.

## Density and layout

UI density controls trade whitespace for information. Sidebar sections can be toggled (hide the standalone Chats list or the Workspace tab), and settings are deep-linkable from sidebar search.

## Shape the provider surface

- **Order**: drag providers into a custom order in settings — the composer, sidebar, and search palette follow it.
- **Favorites**: star models in the picker to float your picks above large Codex/Cursor/OpenCode catalogs.
- **Custom binaries**: point Synara at your own provider executable (a specific Codex or Claude Code install) instead of the default path.
- **Usage panels**: Codex, Claude, and Cursor usage limits and pace appear in the chat Environment panel and settings.
- **Update checks**: background provider CLI version checks can be disabled with the `enableProviderUpdateChecks` setting.

## Keyboard shortcuts

A settings panel covers shortcut exploration and customization; power users can go further with the [keybindings JSON file](/docs/reference/keyboard-shortcuts).

## Profile and activity stats

A local profile tracks your patterns: most-worked project, provider and model mix, active hours, and a token heatmap. It stays on your machine, like the rest of the workspace data.

## Releases, in the app

After an update, a one-time "What's new" dialog summarizes the release, and the full history lives under Settings → Release history. Update downloads show progress on the sidebar update button, installs self-verify after restart, and if an in-app update fails you can always fall back to the [download page](/install).
